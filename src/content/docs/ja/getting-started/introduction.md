Appy Stamp では、ロイヤルティプログラムの上に開発する 3 つの方法を提供しています。コードが動作する場所に合うものを選択してください。

## JavaScript SDK

ストアフロントや任意のウェブサイトでカスタムのロイヤルティ UI を構築するための、軽量なブラウザスクリプト（3KB 未満、依存関係なし）です。ログイン中の顧客にスコープされるため、ストアフロントウィジェット、カスタムリワードページ、ヘッドレスストアフロントに適しています。すべての有料プランで利用できます。

[SDK インストール](/ja/sdk/installation/)から始めましょう。

## REST API

マーチャントとして認証し、ショップ内の任意の顧客に対して操作できる、サーバー間の HTTP API です。バックエンド、CRM、自動化ツールから使用します。現在クローズドベータで、有料プランで利用できます。

[REST API 概要](/ja/rest-api/overview/)から始めましょう。

## GraphQL API

REST API と同じデータと機能を、型付きでイントロスペクション可能な単一エンドポイントを通じて提供します。必要なフィールドだけを 1 回のリクエストで正確に取得できます。現在クローズドベータで、有料プランで利用できます。

[GraphQL API 概要](/ja/graphql-api/overview/)から始めましょう。

## 扱えるもの

3 つの連携方法すべてで、以下にアクセスできます。

- ショップ設定（プログラム名、スタンプブランディング、通貨）
- 獲得ルール、リワード、VIP ティア
- 顧客（スタンプ残高、VIP ティア、アクティビティ履歴を含む）
- 獲得済みリワードコードと交換
- スタンプの付与と調整（REST と GraphQL）
